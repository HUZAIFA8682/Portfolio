ForeverBook

AI-Driven Episodic Story Generation Platform

Project Overview

ForeverBook is a backend-first AI storytelling platform that automatically generates full fantasy books, structured into chapters and episodes, while maintaining narrative continuity across generations.


The system goes beyond simple text generation by introducing story planning, episodic arcs, chapter summaries, and AI-generated book and chapter titles. It is a complete storytelling engine designed for scalability and structure.

The Problem & Solution

THE CHALLENGE

Manually writing long-form fantasy stories breaks continuity, doesn’t scale, and is hard to structure into cohesive chapters. Most AI tools just generate disconnected streams of text.

THE SOLUTION

  • ✨ Pre-Planning: Structures the story arc before writing a single word.
  • ✨ Strict Boundaries: Enforces episode and chapter limits.
  • ✨ Memory: Uses previous chapter summaries to guide future generation.

System Architecture

The system uses a sophisticated 5-step pipeline to ensure quality and continuity, processing requests asynchronously to handle long-running AI generation tasks.

1. User Input

User creates a Story Plan. The system initializes the world and setting.

2. Structural Generation

System generates a Story Arc and a 24-episode chapter plan.

3. AI Intelligence

AI generates the Book Title and unique Chapter Titles based on context.

4. Async Processing

Episodes are generated asynchronously in the background (non-blocking).

5. Continuity Engine

Summaries are stored in DB and fed into the next chapter's prompt.

Key Features

  • ✅ Contextual AI Titles: AI-generated book and chapter titles using contextual prompts.
  • ✅ Dynamic Summaries: Chapter titles dynamically generated using previous chapter summaries.
  • ✅ Structured Planning: Automatic 24-episode chapter planning.
  • ✅ Async Execution: Background task–based episode generation (non-blocking).
  • ✅ Story Continuity: Persistent story continuity using database-stored summaries.
  • ✅ Fault Tolerance: Safe fallback mechanisms for AI failures.
  • ✅ Secure Access: Token-controlled access to story generation.
  • ✅ Modern Architecture: REST-based FastAPI architecture with PostgreSQL.
